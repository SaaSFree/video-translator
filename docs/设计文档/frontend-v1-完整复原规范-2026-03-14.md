# Frontend V1 完整复原规范

## 1. 文档目标

本文件是当前前端的复原标准。

即使下列文件全部丢失，也应当能仅凭本文恢复当前前端：

- `frontend/index.html`
- `frontend/app.js`
- `frontend/styles.css`

本文只描述“当前代码已经存在的行为”，不保留旧 demo 结构或已经废弃的布局节点。

## 2. 页面定位

当前前端是一个三栏工作台：

1. 左侧 `Sidebar`
2. 中间 `Source 工作台`
3. 右侧 `Target 工作台`

页面服务于 3 条主操作：

1. `转写分段`
2. `翻译合成`
3. `转写合成`

## 6. 模型设置区

### 6.1 顺序

设置项顺序固定为：

1. `语音转写`
2. `语音合成`
3. `文本校译`

### 6.2 选项

`语音转写`：

- `MLX Qwen ASR / 0.6B-8bit`
- `MLX Qwen ASR / 1.7B-8bit`

`语音合成`：

- `MLX Qwen TTS / 0.6B-Base-8bit`
- `MLX Qwen TTS / 1.7B-Base-8bit`

`文本校译`：

- `Codex / gpt-5.4 / none`
- `Codex / gpt-5.4 / low`
- `Codex / gpt-5.4 / medium`
- `Codex / gpt-5.4 / high`
- `Codex / gpt-5.4 / xhigh`

### 6.3 默认值

按当前代码恢复：

- `asr_model = mlx-community/Qwen3-ASR-1.7B-8bit`
- `tts_model = mlx-community/Qwen3-TTS-12Hz-1.7B-Base-8bit`
- `review_backend = codex-medium`

### 6.4 重绘规则

模型设置区不能在每次轮询时无脑重绘。

必须保留“只在配置真的变化时重绘”的机制，否则用户点下拉时会被轮询替换 DOM，出现：

- 下拉立刻收起
- 值跳回去
- 选择失败

## 7. 状态卡

### 7.1 第一层文案

顶部阶段文案必须优先显示：

- `空闲（0/0）`
- `正在转写（x/y）`
- `正在纠错（x/y）`
- `正在翻译合成（x/y）`
- `正在重新合成（x/y）`
- `正在时序对齐（x/y）`
- `已完成`

### 7.2 第二层文案

`#job-message` 为摘要标题，例如：

- `就绪`
- `翻译合成已完成`
- `转写分段失败`

### 7.3 长错误处理

状态卡只能显示摘要，不得直接显示完整 traceback。

完整错误只允许保留在 tooltip 或后端 `error_detail`，避免把左侧状态卡撑爆。

## 8. 项目列表

### 8.1 卡片内容

每个项目卡至少显示：

- 项目名
- 状态摘要
- 更新时间

### 8.2 右键菜单

当前顺序固定为：

1. `停止任务`
2. `导出项目`
3. `显示纠错`
4. `隐藏纠错`
5. `截取片段`

## 9. Source 工作台

### 9.1 控制区

source 控制区固定包含：

- 播放
- 暂停
- 停止
- 重放
- 倍速
- 音量
- 语言
- `转写分段`

### 9.2 条目结构

每条 source 条目由三部分组成：

1. 左侧操作列
2. 中间 `textarea`
3. 可选的 `Codex 纠错面板`

左侧操作列固定包含：

- 段号
- 时间范围
- 状态
- `播放`
- `保存`
- `试听`

### 9.3 编辑规则

source 文本可直接编辑。

保存按钮规则：

- 未改动时灰色禁用
- 文本与当前已保存值不同才启用

`试听` 按钮规则：

- 仅当存在 `reference_audio_path` 时启用
- 播放的是处理后的 source 参考音频，而不是 source 播放切片

### 9.4 纠错面板

source 纠错提示显示在条目内部。

待确认态只保留：

- `接受`
- `拒绝`
- `全部待确认项` 勾选框

不再保留顶部全局批量按钮。

## 10. Target 工作台

### 10.1 控制区

target 控制区固定包含：

- 播放
- 暂停
- 停止
- 重放
- 倍速
- 音量
- 语言
- `翻译合成` / `重新合成`

### 10.2 目标视频

右侧视频必须始终可见。

规则：

1. 若已有 `target/dubbed.v1.mp4`，优先播放 target 视频
2. 若还没有最终 target 视频，则回退显示 source 视频画面
3. 在 fallback 期间，点击 target 段播放时需要做到“原视频画面 + 当前 target 音频预览”联动

### 10.3 右侧占位条目

当左侧已有 source 条目时，右侧必须立即生成相同数量的 target 条目骨架。

规则：

1. 左侧有多少条，右侧就先渲染多少条
2. 已完成的 target 条目显示真实文本
3. 未生成的条目显示空文本与 `待生成`
4. 这样用户在 target 生成过程中滚动页面，左右条目仍能保持对齐

### 10.4 Target 编辑

target 文本同样可以直接编辑，并带 `保存` 按钮。

保存按钮规则与 source 相同：

- 未改动时禁用
- 有真实改动后启用

### 10.5 重新合成

当 target 文本存在已保存修改时：

- 主按钮文案切换为 `重新合成`

点击后只执行：

1. 重新合成语音
2. 重新对齐
3. 重新拼接 track
4. 重新生成 target 视频

不会重新翻译文本。

## 11. 左右同步规则

### 11.1 条目数量

左右数量始终以 source 为骨架保持一一对应。

### 11.2 条目高度

对应条目的高度必须保持完全一致。

实现上应当按 `segment_id` 做双侧高度同步，而不是让两列各自自由撑高。

### 11.3 滚动同步

两侧字幕区滚动需要同步。

必须满足：

1. 触摸板快速滚动时也能跟上
2. 当前操作的一侧是 leader
3. 被同步的一侧不能反向把 leader 拉回去
4. 刷新后滚动位置恢复
5. 若当前处于编辑态，刷新后也要恢复焦点和光标

## 12. 播放与定位

### 12.1 整体视频播放

点 source / target 视频区的播放按钮时：

- 播放对应视频
- 文本高亮与滚动随时间更新

### 12.2 段内播放

点条目 `播放` 时：

- 跳到该段开始时间
- 自动滚动到对应条目
- 另一侧若有对应条目，也跟随定位

### 12.3 播放时自动滚动

播放进行中，两侧字幕区都要跟随当前活动段自动滚动。

## 13. 文本编辑状态恢复

刷新页面时必须保留：

1. 若当前未编辑，则恢复左右字幕区滚动位置
2. 若当前正编辑某个 `textarea`，则恢复：
   - `projectId`
   - `side`
   - `segmentId`
   - 光标起止位置
   - 文本框内部滚动位置

## 16. 当前已知边界

前端当前仍属于 V1，已知边界如下：

1. target 文本与音频已经可编辑、可重合成，但 target 音质层面仍存在后端待优化项
2. 状态栏已做摘要化，但不会在前端展开完整错误详情
3. 右侧占位条目解决的是对齐问题，不代表 target 内容已完成

# Source V1 转写分段设计

## 1. 文档目的

本文定义当前代码里的 `转写分段` 实现，而不是理想化草案。

目标是说明：

1. 输入与输出
2. 真实阶段流转
3. 文件落盘
4. 与 source review 的关系
5. 与 `转写合成` 一键流程的边界

## 2. 目标

`转写分段` 的目标是产出可校对的 source 工作稿，而不是最终润色正文。

当前实现必须满足：

1. 按语音间隙优先切 ASR 块
2. 生成带时间戳的 source 段
3. 生成逐段播放音频
4. 生成 source 纠错建议
5. 不自动覆盖 source 正文

## 3. 输入

项目级输入：

- `source/original.mp4`
- `source_language`
- 当前系统 `asr_model`
- 当前系统 `review_backend`

当前 `review_backend` 选项为：

- `codex-none`
- `codex-low`
- `codex-medium`
- `codex-high`
- `codex-xhigh`

兼容说明：

- 遗留的 `codex-minimal` 会在运行时迁移为 `codex-none`

## 4. 模型默认值

按当前代码默认值：

- `asr_model = mlx-community/Qwen3-ASR-1.7B-8bit`
- `review_backend = codex-medium`

## 5. 总流程

当前 `转写分段` 实际执行：

```text
导入视频
  -> 提取 source/original.wav
  -> 按语音间隙切 ASR 块
  -> 逐块转写
  -> 汇总 transcript items
  -> 生成 source segments
  -> 进行 source gap repair
  -> 写 source.v1.json / source.v1.srt
  -> 切 source 播放音频
  -> 切 source 参考音频
  -> 运行 Codex source review
  -> 等待人工确认
```

## 6. 阶段说明

### 6.1 提取原音频

输入：

- `source/original.mp4`

输出：

- `source/original.wav`

### 6.2 ASR 切块转写

规则：

1. 优先按静音窗口切块
2. 默认参数：
   - `target = 24s`
   - `max = 28s`
   - `min = 8s`
   - `min_silence = 0.5s`
   - `noise = -32dB`
3. 找不到合适静音时才回退到固定切块

状态：

- `transcribing`

常见状态文案：

- `正在转写（x/y）`
- `Running MLX ASR recognition (x/y chunks).`

### 6.3 生成 source 段

source 段来源于 transcript，而不是 Codex 改写。

这一步只做最小结构整理：

- `id`
- `index`
- `start`
- `end`
- `text`

### 6.4 Source gap repair

当前实现包含 gap 修复。

如果相邻 source 段之间存在明显的漏转写空洞，会尝试回收 gap 内的漏掉内容，再重新规范化段序。

这是 source 阶段的一部分。

### 6.5 切 source 音频

当前会写出两类 source 音频：

1. 播放片段：
   - `voices/source-segments/<segment_id>.wav`

2. 参考片段：
   - `voices/source-reference-segments/<segment_id>.wav`

其中：

- `source-segments` 是精确按段播放的切片
- `source-reference-segments` 是带少量上下文的参考片段

### 6.6 Source review

当前 source review 是 `Codex` 逐段建议层：

1. 逐段读取 source 文本
2. 只指出可能的 ASR 错误
3. 不自动改正文
4. 用户接受 / 拒绝 / 自定义保存后，source 正文才改变

落盘文件：

- `jobs/source_correction_review.json`

状态：

- `reviewing`

## 7. 产物

当前 source 链路核心产物如下：

- `source/original.wav`
- `segments/source.v1.json`
- `subtitles/source.v1.srt`
- `voices/source-segments/*.wav`
- `voices/source-reference-segments/*.wav`
- `jobs/source_correction_review.json`

## 8. 数据边界

### 8.1 `转写分段`

只负责：

- source ASR
- source 段结构
- source 播放音频
- source review 建议

### 8.2 不负责

不负责：

- target 翻译
- target 合成
- target 对齐
- target 视频输出

## 9. 人工确认规则

source review 的人工动作只有 3 类：

1. `接受`
2. `拒绝`
3. 自定义修改后 `保存`

改动会同步写回：

- `segments/source.v1.json`
- `subtitles/source.v1.srt`
- `jobs/source_correction_review.json`

## 10. 与一键流程的关系

### 10.1 单独点击 `转写分段`

行为：

1. 清空 source 产物和旧 review
2. 重跑 source 链路
3. 停在 source review，等待人工确认

### 10.2 点击 `转写合成`

行为：

1. 清空 source + target 产物
2. 先跑 source 链路
3. 若 source review 还有待确认项，则自动接受全部修订
4. 继续进入 target 链路

也就是说：

- source review 在 `转写分段` 模式下是人工门槛
- 在 `转写合成` 模式下则会自动清过这道门

source 链路相关文件位于：

- `jobs/runtime.json`
- `logs/events.jsonl`

## 12. 当前已知边界

当前 source 链路仍有这些边界：

1. source 分段仍然主要是工程分段，不是深度语义分句
2. source-reference-segments 仍然是简单参考音频，不是经过声学质量筛选的专用 TTS 参考清单
3. source review 目前是一段一段独立建议，不会做跨段联合校正

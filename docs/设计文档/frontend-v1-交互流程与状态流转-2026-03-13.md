# Frontend V1 交互流程与状态流转

## 1. 当前唯一主流程

```text
导入视频
  -> 选择项目
  -> 点击“转写分段”
  -> 提取源音频
  -> 按语音间隙分段转写
  -> 生成 source 分段文本
  -> 切出逐段音频
  -> Codex 逐段纠错
  -> 用户确认
```

## 2. 用户主操作流程

### 2.1 导入视频

1. 用户点击 `导入视频`
2. 选择 mp4 文件
3. 前端上传原视频并创建项目
4. 项目出现在左侧列表
5. 页面自动切换到该项目

### 2.2 启动转写分段

1. 用户点击 `转写分段`
2. 若已有旧结果，弹确认
3. 后端清空旧输出
4. worker 启动 source pipeline
5. 前端轮询项目详情并刷新状态

### 2.3 查看转写结果

1. 原视频出现在播放器中
2. source 段列表出现
3. 每段可以播放切段音频
4. 每段可以定位回原视频时间点

### 2.4 查看纠错建议

1. 转写分段完成后，Codex review 继续逐段执行
2. 纠错完成的段会出现建议卡
3. 若项目设置为隐藏纠错，则正文仍显示，但建议卡不显示

### 2.5 确认纠错

用户有三种处理方式：

1. 点击 `接受`
2. 点击 `拒绝`
3. 直接改正文后点击 `保存正文`

第三种方式本质上是“自定义采用”。

## 3. 右键菜单流程

### 3.1 停止任务

1. 用户右键项目
2. 点击 `停止任务`
3. 弹确认框
4. 确认后终止 worker
5. 状态变为停止/空闲

### 3.2 显示纠错 / 隐藏纠错

1. 用户右键项目
2. 选择 `显示纠错` 或 `隐藏纠错`
3. 前端仅切换建议层的显示状态
4. source 正文不改变

### 3.3 清空项目

1. 用户右键项目
2. 点击 `清空项目`
3. 弹确认框
4. 保留原始视频与配置
5. 删除所有处理产物

### 3.4 删除项目

1. 用户右键项目
2. 点击 `删除项目`
3. 弹确认框
4. 删除整个项目目录

### 3.5 截取片段

1. 用户右键项目
2. 点击 `截取片段`
3. 输入开始/结束时间
4. 后端裁出一个新项目
5. 新项目出现在列表中

### 3.6 导出项目

1. 用户右键项目
2. 点击 `导出项目`
3. 下载整个项目 zip

## 4. 状态流转

### 4.1 项目状态

```text
idle
  -> extracting_audio
  -> transcribing
  -> reviewing
  -> source_review
  -> source_ready / edited
```

### 4.2 状态说明

1. `idle`
   - 项目存在，但还没开始或已被清空

2. `extracting_audio`
   - 正在从原视频提取 source 音频

3. `transcribing`
   - 正在按语音间隙分块做 ASR
   - 同时准备 source 片段音频

4. `reviewing`
   - Codex 正在逐段生成纠错建议

5. `source_review`
   - Codex 已跑完，仍有待确认项

6. `source_ready`
   - source 文本准备完成，且没有待确认项

7. `edited`
   - 用户已经通过接受或自定义修改改动了正文

## 5. 顶部状态区文案规则

### 5.1 运行中

- `正在提取音频（x/y）`
- `正在转写（x/y）`
- `正在纠错（x/y）`

### 5.2 待人工处理

- `待确认（x/y）`

这里的 `x/y` 表示已处理建议数 / 总建议数。

### 5.3 完成与失败

- `已完成（x/y）`
- `失败（x/y）`

## 6. 纠错确认流转

### 6.1 单条接受

`pending -> accepted`

同时：

- source 正文替换为建议文本
- 该段 status 变为 `edited`

### 6.2 单条拒绝

`pending -> rejected`

同时：

- source 正文保留原文

### 6.3 自定义保存

`pending -> customized`

同时：

- source 正文替换为用户输入文本

### 6.4 批量作用

当勾选 `全部待确认项` 时：

1. 用户点某一条建议上的 `接受`
2. 系统将所有 `pending` 项全部接受

点 `拒绝` 同理。

## 7. 前端轮询规则

前端持续轮询当前项目详情与项目列表。

轮询目的：

1. 更新运行状态
2. 更新 source 段列表
3. 更新 Codex 纠错进度
4. 更新项目状态标签

## 8. 已删除的旧交互流

以下流程已彻底移出当前产品范围：

- 翻译合成
- 转写合成
- target 结果查看
- source / target 对照确认
- 完整流程一键执行

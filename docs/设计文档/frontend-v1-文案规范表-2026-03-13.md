# Frontend V1 文案规范表

## 1. 总原则

当前前端文案遵循以下规则：

1. 只说 source 链路，不说 target。
2. 只说转写、纠错、确认，不说翻译、合成、完整流程。
3. 任务态尽量短、明确、可扫描。
4. 第一层状态尽量保持 `阶段（x/y）` 格式。

## 2. 按钮文案

| 位置 | 默认文案 | 运行中文案 |
| --- | --- | --- |
| Sidebar 上传 | `导入视频` | 无 |
| Source 控制区 | `转写分段` | `正在转写分段...` |
| 分段卡片 | `播放片段` | 不变 |
| 分段卡片 | `定位视频` | 不变 |
| 分段卡片 | `保存正文` | 不变 |
| 建议卡 | `接受` | 不变 |
| 建议卡 | `拒绝` | 不变 |
| 截取弹窗 | `确认截取` | 无 |

## 3. 项目右键菜单文案

固定顺序如下：

1. `停止任务`
2. `导出项目`
3. `显示纠错`
4. `隐藏纠错`
5. `截取片段`
6. `清空项目`
7. `删除项目`

## 4. 第一层状态文案

| 状态 | 文案格式 |
| --- | --- |
| 空闲 | `空闲（0/0）` |

| 纠错 | `正在纠错（x/y）` |
| 待确认 | `待确认（x/y）` |

| 失败 | `失败（x/y）` |

## 6. 空状态文案

| 场景 | 文案 |
| --- | --- |
| 没有项目 | `还没有项目，先导入一个视频。` |
| 未选项目 | `请选择左侧项目，或先导入一个视频。` |
| 未转写 | `还没有 source 文本。点击“转写分段”开始处理。` |
| 无纠错摘要 | 不显示摘要栏 |

## 7. 纠错摘要栏文案

### 7.1 显示纠错时

- `Codex 纠错`
- `当前显示纠错提示`
- `全部待确认项`
- `勾选后，点击任意一条建议上的“接受”或“拒绝”，会作用到当前所有待确认项。`

### 7.2 隐藏纠错时

- `当前已隐藏纠错提示`
- `右键项目并选择“显示纠错”，即可恢复逐条建议面板。`

## 8. 建议卡状态标签

| 内部状态 | 文案 |
| --- | --- |
| `queued` | `排队中` |
| `processing` | `处理中` |
| `pending` | `待确认` |
| `accepted` | `已接受` |
| `rejected` | `已拒绝` |
| `customized` | `已自定义` |
| `unchanged` | `无需修改` |
| `failed` | `失败` |

## 9. 确认框文案

### 9.1 停止任务

```text
确认停止「项目名」的当前任务？
当前进行中的转写、纠错或切段都会被终止。
```

### 9.2 清空项目

```text
确认清空「项目名」吗？
会保留项目配置和原始视频，其余产物都会删除。
```

### 9.3 删除项目

```text
确认删除「项目名」吗？
项目目录会被彻底删除。
```

### 9.4 重新转写

```text
重新执行会清空当前转写结果、切段音频和纠错记录，确定继续吗？
```

## 10. 当前禁用的旧文案

以下文案在当前版本中不应再出现：

- `翻译合成`
- `转写合成`
- `开始完整流程`
- `请先完成转写`
- `请先转写分段`
- `target`
- `目标语言`
- `语音合成`

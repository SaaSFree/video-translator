# Frontend V1 界面与交互设计

## 1. 文档定位

- 项目目录：`/Volumes/8TR0/codex/video_translater`
- 文档日期：2026-03-13
- 当前范围：仅覆盖 source 侧转写与纠错确认

本版前端不再承载翻译、配音、目标视频或完整流程设计。页面只服务于 source 文本生产与校对。

## 2. 核心目标

当前前端围绕以下目标设计：

1. 用户导入视频后，能稳定地执行 `转写分段`。
2. 用户能直接看到按语音间隙生成的 source 文本与逐段音频。
3. Codex 的纠错建议必须逐段可见、可接受、可拒绝、可人工直接改正文。
4. 用户能明确区分：
   - 原始 source 正文
   - Codex 纠错建议
   - 最终人工确认结果

## 3. 总体布局

页面使用 `左侧工具栏 + 右侧单工作台` 布局。

### 3.1 左侧工具栏

左侧固定承担三类职责：

1. 模型设置
2. 当前任务状态
3. 项目浏览与导入

### 3.2 右侧工作台

右侧只保留一个 source 工作台，不再显示 target 面板。

工作台内从上到下包括：

1. 原视频播放器
2. 播放控制条
3. 纠错摘要栏
4. source 分段文本列表

## 4. 视觉方向

当前视觉风格采用“纸面工作台”方向：

- 浅暖底色
- 半透明卡片
- 圆角较大
- 以棕红作为主强调色
- 以绿、黄、红区分完成、待确认、失败状态

这个方向的目的不是做“科技感仪表盘”，而是让用户把它当成一个稳定的校对台。

## 5. 模型设置区

模型设置区只保留两项：

1. `语音转写`
2. `Codex纠错`

不再出现以下配置：

- 文本翻译
- 语音合成
- 目标语言
- 完整流程相关配置

每个设置项采用：

- 一行标题
- 一个下拉框
- 一段可选说明文字

## 6. 状态区

状态区使用一个进度轨道卡片承载。

### 6.1 第一层状态

第一层状态必须始终是“阶段 + x/y”格式。

允许的主要文案：

- `空闲（0/0）`
- `正在提取音频（x/y）`
- `正在转写（x/y）`
- `正在纠错（x/y）`
- `待确认（x/y）`
- `已完成（x/y）`
- `失败（x/y）`

### 6.2 第二层状态

第二层状态用于展示当前任务的简短中文说明，例如：

- 正在准备转写，共 86 个语音块
- 正在识别第 12/86 个语音块
- 正在切出第 34/254 段音频
- Codex 纠错建议已生成

### 6.3 辅助信息

状态卡底部保留三项辅助信息：

- `本级 x/y`
- `已耗时`
- `预计剩余`

## 7. 项目列表

项目列表承载以下能力：

1. 展示项目名称
2. 展示项目当前状态
3. 展示简短辅助信息
4. 支持单击切换项目
5. 支持右键菜单操作

每个项目卡片至少显示：

- 项目名
- 当前状态标签
- 简短说明

说明优先级：

1. 若有待确认纠错：显示 `x 条纠错待确认`
2. 若已有 source 段：显示 `x 段 source 文本`
3. 若仍未处理：显示 `等待转写`

## 8. 项目右键菜单

项目右键菜单保留以下条目：

1. `停止任务`
2. `导出项目`
3. `显示纠错`
4. `隐藏纠错`
5. `截取片段`
6. `清空项目`
7. `删除项目`

设计要求：

- 菜单文字居中
- 菜单项单列展示
- 危险操作单独高亮

### 8.1 停止任务

必须弹确认框。

### 8.2 清空项目

保留：

- 项目配置
- 原始视频

清除：

- 转写结果
- source 音频切段
- subtitle 文件
- Codex 纠错记录
- 导出包

### 8.3 显示纠错 / 隐藏纠错

这两个动作只控制“纠错提示面板是否显示”，不应修改正文。

## 9. Source 工作台

### 9.1 视频播放器

顶部播放器始终展示原视频，不再出现目标视频播放器。

### 9.2 控制条

控制条包含：

1. 播放
2. 暂停
3. 停止
4. 重放
5. 倍速选择
6. 音量
7. source 语言
8. `转写分段`

`转写分段` 是当前唯一流程入口。

## 10. 纠错摘要栏

纠错摘要栏位于控制条下方。

它用于展示：

- 当前是否显示纠错
- 纠错统计
- “全部待确认项”勾选框

该区域不再承载批量接受/批量拒绝按钮。

批量操作规则改为：

1. 用户勾选 `全部待确认项`
2. 再点击任意单条建议上的 `接受` 或 `拒绝`
3. 系统将该动作应用到所有待确认项

## 11. Source 分段卡片

每个分段卡片包含：

1. 段号
2. 时间范围
3. 当前正文状态
4. 正文文本框
5. `播放片段`
6. `定位视频`
7. `保存正文`
8. 可选的纠错建议卡

### 11.1 正文文本框

正文文本框始终显示当前 source 正文。

这里的文本可能来自：

- 原始 ASR
- 接受纠错后的正文
- 用户自定义修改后的正文

### 11.2 播放片段

播放的是当前 segment 对应的切段音频，而不是整段视频。

### 11.3 定位视频

将原视频定位到该段开始时间，并开始播放。

### 11.4 保存正文

如果该段存在 `pending` 纠错建议，则“保存正文”按自定义修改处理。
如果没有 `pending` 建议，则“保存正文”只更新 source 正文。

## 12. 纠错建议卡

当项目处于“显示纠错”状态时，正文卡片下方显示建议卡。

建议卡包含：

1. 状态标签
2. 原文
3. 建议文本
4. 结构化修改项
5. `接受`
6. `拒绝`

说明：

- 不再在顶部浮层展示批量按钮
- 不再使用遮挡正文的操作层
- 所有可点击动作都放在建议卡内部

## 13. 空状态设计

需要覆盖以下空状态：

1. 没有项目
2. 选中了项目但还没转写
3. 没有纠错建议

空状态文案要求短、明确、不说教。

## 14. 响应式策略

桌面端是主设计目标。

当宽度较小时：

- 左右布局切换为上下布局
- 控制条允许换行
- 分段卡片仍维持单列

当前版本不追求移动端深度优化，但必须保证可用。

## 15. 当前版本不做的内容

以下内容明确不在本版设计范围内：

- 文本翻译 UI
- 语音合成 UI
- target 结果面板
- 双栏对照滚动
- 完整流程按钮
- 自动进入目标视频处理

## 16. 结论

当前前端 V1 已经从“翻译工作台”收口为“source 校对工作台”。

它的职责只有三件事：

1. 把转写结果展示出来
2. 把 Codex 纠错建议展示出来
3. 让用户尽快做确认和改稿
